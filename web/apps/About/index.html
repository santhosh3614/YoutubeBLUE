Youtube BLUE

Youtube BLUE is a conceptual engineering exercise to demonstrate architecture and engineering ability. Although this product does connect to the "Youtube Data API V3", This is not a real venture and not commercially or publicly available to users, but limited to the reviewers, Banno Inc.

Architecture

Youtube BLUE is powered by a simple framework based on the original Javascript fundamentals of native code structuring using the powerful namespace() method to define Classes, Inheritance-chains, Mixin Traits, and most importantly custom HTML Component Classes.

The structural design of the code is different from several apis and frameworks you've already seen. This framework has been refined for over 12 years whereas some of the newer frameworks have just began their life and experience.

Code-named, 'Rhino': Rhino is based on the idea of following w3c specs and recommendations when possible. The idea is to build a UI Toolkit that will limit the learning curve and leverage the existing w3c documentation.

Introducing the "Rhino Engine"

Youtube BLUE was graphically designed, spliced, coded and programmed in 3 1/2 full days (approx: 27hrs total development time) including learning-curve time for YouTube Data API. All while supporting several major features, bug fixing, server deployment and git-work.

The Rhino Engine is over 12 years old, it came out of IBM in early 2000 and was matured later at WeightWatchers.com and Situated Research LLC, an in-house R&D firm by a group of enthusiasts. It has silently laid in the sun under the trees, bathing in mud and grunting with anticipatation to be revealed as a true beast for Mobile/Web app development.

Engineered from the ground up, Rhino uses simple and limited parts in it's engine. It offers just one core method out of the box called namespace() -- where Classes are forged and inheritance-chains are established to share commonality within families and traits as reusable modules across familes of Classes. A true prototypal engine. It can be used to model simple objects without UI's such as a "Bank Transfer" (DAO) or things requiring a UI such as a customized Toggle Switch (custom HTML Component) or even a Warrior character Class for overhead/sidescrolling 2D game development.

Rhino does not rely internally on binders or controllers or models but transparently embeds the idea of MVC into a single object and allows the developer to rely on skill to achieve sophistication when desired, so there are no memory leaks or silently burning CPU cycles.

Rhino relies on pure HTML5, CSS3 and native Javascript coding experience. Unlike Angular, there are no custom tag/element schemas to learn, no binders, no seperate controllers and views and templates, localization engines, custom tag attributes...no leaks. Rhino is blazing fast, amazingly simple, having few parts but the power of an inline-6 to build CRUD apps, games and mobile Cordova apps for the Android and iOS app-store all while supporting the ability for those apps to run on the web, from disk, and disconnected from a wifi/lan connection because it relies on data, not URI connections.
